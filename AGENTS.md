# Agent Guidelines — Spark ClickHouse Connector

## What this is

- Spark DataSource V2 connector for ClickHouse.
- Talks to ClickHouse over HTTP via `com.clickhouse:client-v2`. **No JDBC, no
  V1 client.**
- Public surface (frozen names): `com.clickhouse.spark.ClickHouseCatalog`,
  `com.clickhouse.spark.ClickHouseTableProvider`, `ClickHouseTable`, the
  `clickhouse` short name, and the `com.clickhouse.spark` `groupId` / package.

## Supported versions

| | |
|--|--|
| **Spark** | 3.3, 3.4, 3.5, 4.0 |
| **Scala** | 2.12 (Spark 3.3 / 3.4 / 3.5) · 2.13 (all) — Spark 4.0 is 2.13 only |
| **Java** | 8 (Spark 3.3 / 3.4 / 3.5) · 17 (all) — Spark 4.0 requires 17 |

Module layout, per-version API drift, and Java client V2 non-negotiables:
[`.claude/code-layout.md`](.claude/code-layout.md). Read before editing code.

## Build & test

Pick exactly one Spark version per invocation. Use `-D` (system property), NOT
`-P`. Subproject names embed the Scala suffix
(`:clickhouse-spark-<v>_<scala>:`). Docker must be running for tests.

```bash
./gradlew clean build -x test -Dspark_binary_version=3.5 -Dscala_binary_version=2.13
./gradlew test          -Dspark_binary_version=3.5 -Dscala_binary_version=2.13
./gradlew spotlessApply -Dspark_binary_version=3.5
```

Run spotless **for every Spark version you touched**; CI fails otherwise.

Read [`.claude/build-and-test.md`](.claude/build-and-test.md) when: the build
fails unexpectedly (daemon / profile-switch, stale classes); you need the fat
jar; you're writing tests (especially Cloud — SharedMergeTree consistency);
you're running `slowTest` / `cloudTest` / a custom `CLICKHOUSE_IMAGE`; you
need to know where a new test file goes; or you used a Scala feature and want
to confirm it cross-builds for 2.12.

## Hard boundaries

1. **Cross-version duplication is load-bearing.** Mirror changes across every
   `spark-<v>/clickhouse-spark/` tree, or justify the exclusion in the PR.
2. **No V1 Java client, no JDBC, no raw HTTP.** All I/O via
   `com.clickhouse:client-v2` through `NodeClient` / `ClusterClient`.
3. **Public-API names are frozen.** See list under "What this is".
4. **Don't restructure Gradle** or `settings.gradle`'s parameterization.
5. **Adding/changing a `ClickHouseSQLConf` entry requires regenerating the
   golden file** for 3.4 / 3.5 / 4.0 (3.3 has no `ConfigurationSuite`):
   ```bash
   UPDATE=1 ./gradlew test --tests=ConfigurationSuite \
     -Dspark_binary_version=<v> -Dscala_binary_version=<s>
   ```
6. **Don't write to `docs/` by hand.** It's deprecated. Only exception:
   `docs/configurations/*.md` regenerated by `ConfigurationSuite` (#5).
   User-visible behavior changes (new options, changed semantics, new type
   support) still require a docs update — submit it to
   [`ClickHouse/clickhouse-docs`](https://github.com/ClickHouse/clickhouse-docs),
   not the local `docs/`.
7. **Jackson shading is version-specific.** 4.0 bundles + relocates;
   3.3/3.4/3.5 exclude. Exclude transitive Jackson from new deps on all four.
8. **`VariantType` is Spark 4.0-only.** Don't backport to `spark-3.x/`.
9. **Don't leak `AutoCloseable`s.** Use `Utils.tryWithResource` or
   `try`/`finally` on every exit path.
10. **No silent fallbacks on I/O.** Return `false` from `pushXxx(...)` when
    unsupported — never produce SQL ClickHouse will reject at runtime.
11. **Pushable scalar functions in `func/` must match ClickHouse semantics
    exactly** (null handling, integer width, hash seed). Wrong → silent shard
    routing corruption.
12. **Every user-visible change needs a `CHANGELOG.md` entry** under
    `[Unreleased]` in the correct section (`Added` / `Changed` / `Fixed` /
    `Removed`) with a PR link. Breaking changes (removed configs, renamed public classes,
    changed type mappings etc.) go under a **Breaking Changes** sub-section and
    must also be called out in the PR description.
13. **No Scala 2.13-only features in the 3.x trees** — they cross-build for
    2.12. Only 2.12 CI lanes catch this.
14. **Style:** Spotless + scalafmt 3.6.1, `maxColumn=120`, Apache-2.0 header on
    every new source file (RAT enforces).
